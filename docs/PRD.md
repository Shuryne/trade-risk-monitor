# 每日交易订单风险监控系统 — 产品需求文档（PRD）

> **版本**：v2.0  
> **最后更新**：2026-03-22  
> **状态**：待开发

---

## 1. 项目概述

### 1.1 背景

香港证券及期货事务监察委员会（SFC）要求持牌券商建立有效的内部监控系统，识别和报告可疑交易行为，包括市场操纵、内幕交易和洗钱等。对于小型券商而言，购买昂贵的商业化交易监控系统（如 NICE Actimize、Nasdaq Surveillance 等）成本过高，但合规义务不可免除。

目前，工作人员依赖手动翻阅交易记录来发现异常，效率低下且容易遗漏。需要一个轻量级的工具来自动化这一流程。

### 1.2 解决什么问题

| 现有痛点 | 本系统如何解决 |
|---|---|
| 手动逐条审查交易记录，耗时且易遗漏 | 自动化规则引擎批量扫描，秒级完成分析 |
| 缺乏可视化的风险全景视图 | 提供仪表盘，一眼掌握当日风险分布 |
| 风险报告需手动编写，格式不统一 | 一键生成结构化的风险报告，支持导出 |
| 风险判断标准因人而异，缺乏一致性 | 可配置的规则引擎，统一风险判定标准 |
| 无法快速回溯历史风险记录 | 本地持久化存储，支持历史记录查询和趋势分析 |

### 1.3 目标用户

- **主要用户**：券商合规部门工作人员（Compliance Officer），负责每日交易监控和风险报告
- **次要用户**：风控经理（Risk Manager），审阅风险报告并做出决策
- **使用场景**：每个交易日收盘后，从交易系统导出当日订单数据 → 脱敏处理 → 上传到本系统 → 查看分析结果 → 导出报告归档

### 1.4 设计原则

- **数据不出本地**：所有数据处理在浏览器端完成，不上传至任何外部服务器，满足数据安全和合规要求
- **零配置启动**：无需安装数据库、无需后端服务，打开浏览器即用
- **规则可配置**：风险阈值和规则开关可由用户自行调整，适应不同券商的风控策略
- **操作极简**：核心工作流三步完成——上传 CSV → 查看仪表盘 → 导出报告

## 2. 核心功能列表

### 2.1 功能总览与优先级

| 优先级 | 功能模块 | 说明 |
|:---:|---|---|
| P0 | CSV 文件上传与解析 | 系统入口，支持拖拽上传、格式校验、数据预览 |
| P0 | 风险规则引擎 | 核心分析能力，内置多种风险检测规则 |
| P0 | 风险仪表盘 | 分析结果的可视化总览 |
| P0 | 风险订单明细表 | 可筛选、排序的风险订单详细列表 |
| P1 | 风险报告导出 | 生成 PDF 格式的结构化风险报告 |

### 2.2 功能详细描述

#### F1: CSV 文件上传与解析（P0）

**目的**：让用户快速导入当日交易订单数据。

**功能要求**：

- 支持拖拽上传和点击选择文件两种方式
- 支持 `.csv` 格式，文件编码自动检测（UTF-8 with BOM / UTF-8 / GBK）
- 上传后立即解析并展示数据预览（前 10 行），供用户确认数据是否正确
- 自动校验必填字段是否存在，数据类型是否合法（如价格必须为非负数、时间格式是否合法）
- 校验失败时，明确提示哪些行/列存在问题，支持用户修正后重新上传
- 显示数据统计摘要：总行数、时间范围、涉及账户数、涉及标的数、涉及市场

**CSV 原始字段定义**（繁体中文表头，与交易系统导出格式一致）：

```
原始列名           英文映射              类型        说明
─────────────────────────────────────────────────────────────────────
編號              order_id              string     订单唯一编号（原始数据带三重引号包裹，解析时需清洗）
席位號            seat_number           string     席位号（可为空）
創建時間          order_time            string     下单时间，格式为 "YYYY-MM-DD HH:mm:ss TZ"，
                                                  其中 TZ 为 "HKT" 或 "ET"，需统一转换为 HKT
證券賬戶          account_id            string     客户证券账户编号（已脱敏）
買賣方向          side                  enum       交易方向："買入" / "賣出"
開平方向          open_close            enum       开平方向："開倉" / "平倉"
證券代碼          symbol                string     证券代码，如 "0700"（港股）/ "AAPL"（美股）
委託價格          order_price           number     委托价格（正数，精确到小数点后 4 位）
委託數量          order_quantity        number     委托数量（正整数，原始数据可能含千分位逗号如 "10,000"）
剩餘未成交數量     remaining_quantity    number     未成交的剩余数量
已成交數量         filled_quantity       number     已成交数量（原始数据可能含千分位逗号）
已成交金額         filled_amount         number     已成交总金额（原始数据可能含千分位逗号，如 "9,000.0000"）
訂單狀態          order_status          enum       "成交" / "已拒絕" / "已過期" / "已撤單" / "部分成交"
信息              message               string     系统消息/拒绝原因（可为空）
BCAN              bcan                  string     经纪客户编码（Broker-Client Assigned Number），
                                                  港股特有，美股可为空
經紀人號          broker_id             string     经纪人编号
訂單類型          order_type            enum       "限價單" / "競價限價單" / "市價單" / "條件單" 等
訂單種類          order_category        enum       "普通订单" / "长期单"
資產屬性          asset_type            string     资产属性标识，如 "ST"
市場              market                enum       "HK"（港股）/ "US"（美股）
覆盤時間          review_time           string     覆盘时间（可为空）
上手經紀商         clearing_broker       string     上手经纪商名称，如 "華盛"、"ZJ"
上手賬號          clearing_account      string     上手经纪商账号
```

**数据清洗规则**（解析阶段自动执行）：

| 清洗项 | 处理逻辑 |
|---|---|
| UTF-8 BOM | 检测并移除文件开头的 BOM 字节（`\uFEFF`） |
| 订单编号三重引号 | 移除 `編號` 字段值两端的多余引号，提取纯数字字符串 |
| 千分位逗号 | 对 `委託數量`、`已成交數量`、`已成交金額` 等数字字段，移除逗号后转为 number |
| 时区统一 | 将所有时间统一转换为 HKT（UTC+8）：ET 时间 +13 小时（东部标准时间差） |
| 空白值 | 空字符串统一为 `null`，不参与计算 |
| 币种推断 | 根据 `市場` 字段推断币种：HK → HKD，US → USD |
| 成交均价计算 | 新增计算字段：`filled_avg_price = filled_amount / filled_quantity`（未成交时为 0） |
| 委托金额计算 | 新增计算字段：`order_amount = order_price × order_quantity` |

#### F2: 风险规则引擎（P0）

**目的**：自动扫描所有订单，根据预设规则标记潜在风险。

**风险规则定义**：

每条规则包含：规则 ID、规则名称、风险描述、风险等级（HIGH / MEDIUM / LOW）、默认阈值、开关状态。

**内置规则列表**：

| 规则 ID | 规则名称 | 默认风险等级 | 检测逻辑 | 默认阈值 |
|---|---|:---:|---|---|
| R001 | 大额交易检测 | HIGH | 单笔委托金额（order_price × order_quantity）超过阈值。按市场分别设置阈值。 | HK 市场 ≥ 500 万 HKD；US 市场 ≥ 100 万 USD |
| R002 | 高频交易检测 | MEDIUM | 同一账户在 N 分钟内下单次数超过阈值 | 同一账户 5 分钟内 ≥ 10 笔 |
| R003 | 单账户集中度 | MEDIUM | 单一账户的委托总金额占当日全部委托总金额的比例超过阈值 | ≥ 25% |
| R004 | 单标的集中度 | MEDIUM | 单一标的的委托总金额占当日全部委托总金额的比例超过阈值 | ≥ 30% |
| R005 | 成交价格偏离 | HIGH | 成交均价（filled_amount / filled_quantity）偏离委托价格的幅度超过阈值。仅对已成交订单检测。 | 偏离 ≥ 5% |
| R006 | 对敲交易检测 | HIGH | 不同账户在短时间窗口内对同一标的进行方向相反、数量相近的交易 | 时间窗口 2 分钟内，数量偏差 ≤ 10% |
| R007 | 尾盘异常交易 | MEDIUM | 港股市场在收盘前 N 分钟内的大额委托（按 HKT 时间判定）。仅适用于 HK 市场。 | 收盘前 15 分钟（15:45 HKT 后）内单笔委托金额 ≥ 100 万 HKD |
| R008 | Penny Stock 交易 | LOW | 低价股的大额交易。仅适用于 HK 市场（港股仙股较常见）。 | 股价 < 1 HKD 且委托金额 ≥ 50 万 HKD |
| R009 | 撤单率异常 | MEDIUM | 同一账户当日撤单（訂單狀態 = "已撤單"）比例超过阈值 | 撤单率 ≥ 50%（至少 5 笔订单） |
| R010 | 洗售交易检测 | HIGH | 同一账户在短时间内对同一标的先买后卖（或先卖后买），数量相近 | 时间窗口 10 分钟内，数量偏差 ≤ 5% |
| R011 | 被拒订单关注 | LOW | 标记所有被拒绝的订单（訂單狀態 = "已拒絕"），并附带拒绝原因（信息字段）供人工审查 | 无阈值，枚举检测 |

**多市场处理注意事项**：

- 数据中同时包含 HK（港股）和 US（美股）两个市场的订单
- 涉及金额比较的规则（R001、R007、R008）需按市场分别设置阈值和币种
- 集中度计算（R003、R004）按市场分别计算，避免跨币种混合计算导致失真
- 尾盘规则（R007）仅适用于 HK 市场，检测基于 HKT 时间；如未来需覆盖美股尾盘，需增加 ET 时区判断逻辑
- 时间相关规则（R002、R006、R010）使用统一转换后的 HKT 时间进行计算

**引擎执行逻辑**：

1. 加载用户的规则配置（阈值、开关状态）
2. 对原始数据进行预处理：按账户分组、按标的分组、按市场分组
3. 逐条/分组应用每条已启用的规则
4. 每条被标记的订单生成一个风险记录，包含：原始订单数据、触发的规则列表、风险等级（取触发规则中的最高等级）、风险描述
5. 按风险等级排序输出结果

#### F3: 风险仪表盘（P0）

**目的**：提供当日风险的全景视图，帮助工作人员快速判断整体风险状况。

**展示内容**：

- **顶部统计卡片**（5 个）：
  - 当日订单总数
  - 风险订单数量及占比
  - 高风险订单数量
  - 涉及的风险账户数量
  - 按市场分布（HK / US 各多少笔）

- **图表区域**：
  - 风险等级分布（环形图）：HIGH / MEDIUM / LOW 的数量和占比
  - 风险类型分布（水平柱状图）：每条规则触发的次数
  - 交易时间分布（折线图）：按小时分组的订单量（HKT 时间），标注风险订单的叠加
  - 账户风险排名（表格）：触发风险次数最多的 Top 10 账户
  - 市场对比（分组柱状图）：HK / US 市场各自的订单量与风险订单量对比

#### F4: 风险订单明细表（P0）

**目的**：提供风险订单的完整详情，支持深入排查。

**功能要求**：

- 以表格形式展示所有被标记的风险订单
- 支持按以下维度筛选：风险等级、风险规则类型、账户、标的、交易方向、市场（HK / US）、订单状态
- 支持按以下字段排序：委托金额、下单时间、风险等级
- 点击单行可展开详情面板，显示：
  - 触发的所有规则及每条规则的具体参数
  - 同账户当日的所有交易列表（上下文关联）
  - 同标的当日的所有交易列表
  - 订单的信息/拒绝原因（如果有）
- 支持批量标记"已审阅"/"需跟进"/"误报"状态
- 表格支持关键词搜索（搜索订单号、账户、标的、经纪人号）

#### F5: 风险报告导出（P1）

**目的**：生成结构化报告供存档和上报。

**报告内容**：

- 报告封面：报告标题、日期、生成时间、公司名称
- 数据概览：订单总数、各市场交易金额合计、涉及账户数等
- 风险摘要：各等级风险数量、主要风险类型
- 风险订单明细表：完整的风险订单列表
- 附录：当前使用的规则配置参数

**导出格式**：PDF（主要）、CSV（辅助，便于进一步分析）

#### F6: 规则配置面板（P1）

**目的**：让工作人员能根据公司风控策略调整规则参数。

**功能要求**：

- 列出所有内置规则，每条规则可独立开启/关闭
- 每条规则的阈值可修改，修改时提供合理的输入校验和范围提示
- 对多市场规则（R001、R007、R008），支持分市场设置不同阈值
- 提供"恢复默认值"按钮
- 配置变更后，可选择对当前已上传的数据立即重新分析
- 规则配置持久化存储到 localStorage，下次打开时自动加载

#### F7: 历史记录管理（P2）

**目的**：支持回溯查看过去的分析结果。

**功能要求**：

- 每次分析完成后，自动将结果保存到浏览器的 IndexedDB
- 历史记录列表展示：日期、订单数量、风险订单数量、风险概要
- 点击可查看历史分析的完整仪表盘和明细表
- 支持删除单条历史记录
- 存储空间管理：显示已用空间，超过上限时提示清理

#### F8: 趋势分析（P2）

**目的**：发现长期风险模式。

**功能要求**：

- 基于历史记录数据，展示以下趋势图表：
  - 每日风险订单数量趋势（折线图）
  - 各风险类型的触发频次趋势（堆叠面积图）
  - 高频风险账户排名（近 30 日累计）

## 3. 技术栈

### 3.1 技术选型

| 层级 | 技术 | 选型理由 |
|---|---|---|
| **框架** | React 18 + TypeScript | 类型安全，组件化开发，生态成熟 |
| **构建工具** | Vite | 开发体验好，构建速度快 |
| **路由** | React Router v6 | SPA 多页面切换 |
| **状态管理** | Zustand | 轻量级，API 简洁，适合中小型应用 |
| **CSV 解析** | Papa Parse | 业界标准的 CSV 解析库，支持大文件流式解析、编码检测 |
| **图表** | Recharts | 基于 React 的声明式图表库，与 React 集成度高 |
| **UI 组件** | Tailwind CSS + shadcn/ui | 高度可定制的 UI 组件，开箱即用且风格专业 |
| **表格** | TanStack Table（React Table v8） | 高性能虚拟化表格，支持排序、筛选、分页，可处理大数据集 |
| **PDF 导出** | jsPDF + jsPDF-AutoTable | 纯前端生成 PDF，无需后端 |
| **本地存储** | IndexedDB（via Dexie.js） | 浏览器端结构化存储，适合历史数据持久化 |
| **日期处理** | Day.js + timezone 插件 | 轻量级日期库，支持 HKT / ET 时区转换 |

### 3.2 不需要后端的理由

- 所有交易数据已脱敏，但仍为敏感数据，不宜离开本地环境
- 计算量有限（几千条记录的规则匹配），浏览器端完全能胜任
- 无需多用户协作，单机使用即可
- 部署极简：只需一个静态文件服务（Nginx / Caddy / 甚至 `npx serve`）
- 未来如需多用户/集中管理，可在不改变前端的情况下加入后端 API 层

### 4.6 性能考量

- **大数据集处理**：CSV 解析使用 Papa Parse 的流式模式（chunk-based parsing），避免一次性加载到内存导致卡顿
- **表格虚拟化**：TanStack Table 配合虚拟滚动（如 @tanstack/react-virtual），几千行数据流畅渲染
- **规则引擎优化**：对需要分组的规则（高频交易、集中度等），先做一次预处理分组，避免重复遍历
- **Web Worker（可选）**：如果规则引擎计算耗时明显，可将分析逻辑移入 Web Worker，避免阻塞 UI 线程
- **按需渲染图表**：仪表盘图表使用 React.lazy + Suspense 按需加载

**香港市场（HKT, UTC+8）**：

| 时段 | 时间 |
|---|---|
| 开市前时段 | 09:00 – 09:30 |
| 上午持续交易时段 | 09:30 – 12:00 |
| 午间休市 | 12:00 – 13:00 |
| 下午持续交易时段 | 13:00 – 16:00 |
| 收市竞价时段 | 16:00 – 16:10 |

**美国市场（ET, UTC-5 / UTC-4 夏令时）**：

| 时段 | ET 时间 | 对应 HKT |
|---|---|---|
| 盘前交易 | 04:00 – 09:30 | 17:00 – 22:30（前一日） |
| 常规交易 | 09:30 – 16:00 | 22:30 – 05:00（次日） |
| 盘后交易 | 16:00 – 20:00 | 05:00 – 09:00 |

R007（尾盘异常交易）默认以港股 15:45 HKT 之后的交易为检测范围。

### 5.2 时区转换规则

源数据中的时间格式为 `YYYY-MM-DD HH:mm:ss TZ`，其中 TZ 可能是：

- **HKT**：香港时间，UTC+8，无需转换
- **ET**：美国东部时间，需转换为 HKT
  - 标准时间（EST，11月第一个周日 – 3月第二个周日）：ET + 13 小时 = HKT
  - 夏令时间（EDT，3月第二个周日 – 11月第一个周日）：ET + 12 小时 = HKT

转换后统一存储为 ISO 8601 格式并标注为 HKT，同时保留 `original_timezone` 字段标记原始时区。
